# GitHound

GitHound - R Shiny-приложение для анализа GitHub-профилей. Проект объединяет личный кабинет, хранение настроек в ClickHouse, тематический интерфейс, запуск протоколов анализа и архив отчетов.

## Возможности

- Регистрация и вход по почте/паролю.
- Вход через GitHub OAuth.
- Личный кабинет с GitHub-токеном, аватаром и выбором мифологического стиля.
- Темы интерфейса: адская и райская (светлая и темная).
- Мифологические стили: египетский и скандинавский; греческий зарезервирован.
- Протокол Сет: фактический анализ без ИИ.
- Протокол Исида: интеллектуальный анализ.
- Подготовленные ассеты для будущих протоколов Маат.
- Архив отчетов с хранением активных отчетов 7 дней.
- PDF-выгрузка отчетов.
- Поиск уязвимостей через Syft и OSV/MongoDB, если окружение настроено.

## Требования

- R 4.5.x.
- R-пакеты, используемые приложением: `shiny`, `httr`, `jsonlite`, `digest`, `openssl`, `later`, `zip` и пакеты, требуемые функциями анализа.
- ClickHouse с HTTP-доступом.
- Для поиска уязвимостей: MongoDB с OSV-базой и `syft.exe`.
- Для GitHub OAuth: OAuth App в настройках GitHub.

## Настройка окружения

1. Скопируйте пример:

```powershell
Copy-Item .Renviron.example .Renviron
```

2. Заполните `.Renviron` локальными значениями. Файл `.Renviron` не должен попадать в Git.

Минимально нужны:

```text
CLICKHOUSE_HOST=your-clickhouse-host
CLICKHOUSE_PORT=8123
CLICKHOUSE_DB=your-database
CLICKHOUSE_USER=your-user
CLICKHOUSE_PASSWORD=your-password
CLICKHOUSE_HTTPS=false
```

Для GitHub OAuth:

```text
GITHUB_CLIENT_ID=your-github-oauth-client-id
GITHUB_CLIENT_SECRET=your-github-oauth-client-secret
GITHUB_OAUTH_SCOPE=read:user user:email public_repo
```

Для OSV/MongoDB:

```text
OSV_MONGO_URL=mongodb://localhost:27017
OSV_MONGO_DB=osv
OSV_MONGO_COLLECTION=vulns
```

## Запуск

Из корня проекта:

```r

```

Или напрямую:

```r

```

По умолчанию корневой `app.R` запускает приложение на:

```text

```

## ClickHouse

Приложение само создает необходимые таблицы при запуске/использовании:

- `githound_accounts` - аккаунты, GitHub-токены, аватар, стиль интерфейса.
- `githound_remember_tokens` - безопасный remember-token для автологина.
- `githound_report_archive` - архив отчетов.

Важно: пароль пользователя хранится не в открытом виде, а как SHA-256 hash. GitHub-токен сейчас хранится в ClickHouse для использования протоколами анализа. Репозиторий не должен содержать реальные `.Renviron` или секреты.

## Регистрация и требования к паролю

При регистрации пароль должен соответствовать требованиям:

- минимум 10 символов;
- без пробелов;
- минимум одна строчная буква;
- минимум одна заглавная буква;
- минимум одна цифра;
- минимум один специальный символ;
- не быть распространенным слабым паролем;
- не содержать часть email;
- не содержать ник пользователя.

Если пароль слабый, регистрация не создаст аккаунт и покажет причину.

## GitHub OAuth

Для входа через GitHub нужен GitHub OAuth App.

В настройках OAuth App укажите callback URL под адрес, где развернут сайт. Для локальной проверки пример:

```text
http://127.0.0.1:3838/
```

Если приложение запускается на другом порту или домене, callback URL в GitHub должен совпадать с фактическим адресом сайта.

## Протоколы анализа

### Сет

Сухой протокол без ИИ. Собирает фактические данные GitHub:

- профиль;
- коммиты;
- репозитории;
- языки и стек;
- ссылки;
- активность по времени;
- популярность;
- уязвимости зависимостей, если настроены Syft и OSV.

### Исида

Интеллектуальная система интерпретации и детального анализа. Использует более высокий уровень интерпретации отчета.

### Будущие протоколы

В проект уже добавлены изображения для будущих протоколов:

- Маат: `protocol-maat.png`, `protocol-maat-heaven.png`.

## Архив отчетов

Архив хранит отчеты пользователя в ClickHouse. Отчеты видны только пользователю, который их создал.

Правила:

- активный отчет хранится 7 дней;
- после истечения срока получает статус `заархивировано`;
- если создается новый отчет по той же цели и тому же протоколу, старый активный отчет архивируется;
- отчеты можно открыть, скачать в PDF или удалить вручную.

## Темы и стили

Тема:

- рай (светлая);
- ад (темная).

Мифологический стиль:

- египетский;
- скандинавский;
- греческий (пока недоступен).

Выбранный стиль сохраняется в аккаунте и восстанавливается при следующем входе.

## Переменные производительности

Основные настройки параллельности:

```text
GITHOUND_SET_WORKERS=2
GITHOUND_COMMIT_WORKERS=1
GITHOUND_COMMIT_SCAN_WORKERS=16
GITHOUND_SYFT_SCAN_PARALLEL=true
GITHOUND_SYFT_SCAN_WORKERS=4
```

Увеличивайте значения осторожно: GitHub API имеет rate limit, а Syft-анализ может активно нагружать CPU и диск.

## Безопасность

- Не коммитьте `.Renviron`.
- Не храните ClickHouse/GitHub секреты в коде.
- Для production-развертывания используйте HTTPS.
- Для GitHub OAuth настройте callback URL строго под домен приложения.
- Подтверждение email пока не реализовано. 

## Частые проблемы

### `Error sourcing shiny_app/app.R`

Проверьте синтаксис:

```r
parse("shiny_app/app.R")
```

Если ошибка связана с пакетами, убедитесь, что `.Rlibs` подключается и зависимости установлены.

### `Could not connect to localhost port 8123`

ClickHouse недоступен по указанному адресу. Проверьте:

- `CLICKHOUSE_HOST`;
- `CLICKHOUSE_PORT`;
- запущен ли ClickHouse;
- нужен ли `CLICKHOUSE_HTTPS=true`.

### GitHub OAuth возвращает 404

Проверьте `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET` и callback URL в настройках OAuth App. URL должен совпадать с адресом запуска приложения.

### Поиск уязвимостей пропущен

Проверьте:

- путь к `syft.exe`;
- доступность MongoDB;
- переменные `OSV_MONGO_URL`, `OSV_MONGO_DB`, `OSV_MONGO_COLLECTION`;
- наличие загруженной OSV-базы.
